\section{Software Engineering Practices}
\label{sec:software-engineering-practices}

Use this chapter to demonstrate that the project was developed using software engineering practices, not only as an isolated prototype.

Suggested areas to cover:
\begin{itemize}[noitemsep]
  \item Version control and branching strategy.
  \item Team collaboration and task management.
  \item Iterative development process, such as Scrum or agile-inspired work.
  \item Code quality standards.
  \item Code reviews or peer review practices.
  \item Unit testing, integration testing, manual testing, or user acceptance testing.
  \item Continuous integration or deployment pipeline, if used.
  \item Architecture and design decisions.
  \item Documentation practices.
  \item Maintainability and future extensibility.
\end{itemize}
