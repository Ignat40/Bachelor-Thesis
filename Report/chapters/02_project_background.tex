\section{Project Background}
\label{sec:project-background}

This chapter should provide the reader with enough context to understand the project before the detailed problem statement and solution are introduced. Describe the bachelor project, the intended users, and the setting in which the application is expected to be used.

Suggested points to cover:
\begin{itemize}[noitemsep]
  \item The role of repeated practice in speech therapy.
  \item The challenges of maintaining structured home practice.
  \item The potential of mobile applications as supplementary support tools.
  \item The main project deliverables, such as the mobile client, backend, exercise flow, progress tracking, or prototype.
  \item The expected outcome of the project from a software engineering perspective.
\end{itemize}

\section{Project Scope}
\label{sec:project-scope}

Clearly state what is included and what is not included in the project. This helps the reader understand the limits of the work and prevents the report from overclaiming.

\subsection*{Included in Scope}
\begin{itemize}[noitemsep]
  \item Design and implementation of a mobile application prototype.
  \item Guided speech practice exercises.
  \item A child-friendly user experience.
  \item Technical evaluation and testing of the implemented system.
\end{itemize}

\subsection*{Outside Scope}
\begin{itemize}[noitemsep]
  \item Replacing professional speech therapy.
  \item Clinical validation of therapeutic effectiveness.
  \item Diagnosis of speech impairments.
  \item Long-term medical outcome studies.
\end{itemize}
